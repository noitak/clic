# AGENT.md

このリポジトリは `clic` の実装です。
`clic` は **Command Library in Context** の略で、よく使うシェルコマンドを Markdown の runbook として保存し、自然言語で探し、パラメータを埋め、クリップボードへコピーする個人用 CLI です。

## 基本方針

- `clic` は生成したコマンドを実行しません。ユーザーが自分のシェルへ貼り付けるためのコマンドだけを準備します。
- MVP の対象は macOS、zsh、`pbcopy`、Ollama です。不要に他 OS や他 shell へ一般化しないでください。
- 保存データは `~/.config/clic/commands/<id>.md` の Markdown ファイルです。通常のファイル操作で読める、編集できる、同期できる形式を優先します。
- 仕様判断に迷ったら、まず `CLIC_SPEC.md` を正とし、ユーザー向け説明は `README.md` と整合させてください。
- 「検索してコピーする」以上の機能、特にコマンド実行、実行ログ、汎用シェル履歴置換、編集/削除専用サブコマンドは足さないでください。

## 実装メモ

- 言語は Go です。`go.mod` は `module clic`、`go 1.25` です。
- 現状の実装は主に `main.go`、テストは `main_test.go` にあります。小さな CLI なので、既存の関数分割と入出力注入のスタイルに合わせてください。
- CLI の公開サブコマンドは `init zsh`、`add`、`ask`、`list`、`show`、`path` です。未知のサブコマンドは `ask` と同じ検索として扱います。
- `ask` は保存済みエントリから探すだけです。保存済みエントリがない場合に Ollama へ新規コマンド生成を依頼してはいけません。
- `add` の Ollama 生成結果は保存前に検証してください。失敗時は保存しないのが基本です。
- テンプレートの `{{param}}` と `params` の整合性、ID/ファイル名衝突、重複 `original_command`、危険そうなコマンドの確認既定値に注意してください。

## 開発コマンド

```sh
go test ./...
go build -o clic .
```

動作確認で実際のホームディレクトリを汚したくない場合は、一時ディレクトリを `HOME` にして実行してください。

```sh
HOME="$(mktemp -d)" ./clic init zsh
```

## テスト方針

- 変更後は少なくとも `go test ./...` を実行してください。
- CLI の入出力は `stdin`、`stdout`、`stderr` を注入してテストする既存パターンに合わせてください。
- Ollama やクリップボードなど外部依存は、既存の差し替え可能な関数やテスト用環境変数を使って直接依存を避けてください。
- 新しい仕様分岐を足した場合は、ユーザーに見える出力、終了動作、外部依存失敗時の挙動をテストしてください。

## コーディング規約

- 既存のシンプルな標準ライブラリ中心の実装を保ってください。
- フォーマットは `gofmt` に従ってください。
- エラー文は CLI 利用者が次に何をすればよいか分かる短い文にしてください。
- コメントは複雑な意図を補うときだけ追加してください。
- 日本語の文言は README と既存テストに合わせ、ユーザー向け出力を変更する場合はテストも更新してください。

## 安全上の注意

- `clic` 自身が保存または描画したコマンドを実行する機能を追加しないでください。
- 破壊的に見えるコマンドではコピー確認の既定値を `N` にする方針を維持してください。
- `clic` または `command clic` から始まるコマンドを保存しない方針を維持してください。
- ユーザーの実 `~/.config/clic`、実クリップボード、実 zsh 履歴に依存するテストを書かないでください。
